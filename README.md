# Practicasss
Repo para practicar java git y github y en general lo que se aprenda en la clase
## Organization
### Directorio
El repositorio por ahora va a estar organizado en n carpetas siendo n el número de integrantes, cada carpeta lleva su 
nombre y por lo tanto la clase principal de esa carpeta es también el nombre de la persona (crear y subir la carpeta es lo primero que hay que 
hacer al entrar al repo)
### Flujo
En pro de practicar los comandos usuales de git y simular un flujo profesional en github, la forma de subir los avances
a github será creando una rama local donde se va a trabajar, la rama debe iniciar con el prefijo feature/ seguido de lo que
se va a desarrollar (ej.: feature/ejercicio-herencia) asi mismo realizar el push 
## Exercises
El archivo main es donde se subirá los ejercicios que posteriormente usted debe copiar y pegar en su
archivo personal para desarrollarlos
### Tools
Aunque para trabajar es únicamente necesario el JDK y el editor de código, como recomendación es mejor utilizar el JDK 21 
si está usando el JDK 25 averiguar como cambiar a la version 21 (es fácil) por otro lado, en el editor de código recomiendo, por un lado, personalizarlo a su gusto, y por otro usar los siguientes plugins:
- inspection lends (para ver errores antes de ejecutar código)
### Recommendations
A pesar de que esto es solo una práctica entre compañeros de clase me parece esencial acostumbrarse a buenas prácticas de trabajo
es por eso que hay que seguir las siguientes pautas:
- cada commit debe realizar un solo cambio o razón de hacerse (realmente eso es más aplicable más adelante en la clase cuando se arme algún proyecto grande)
- comentar el código (por más sencillo que parezca)
- hacer el código totalmente en inglés (variables, funciones, etc.) y si es de su agrado los comentarios también asi mismo
- como recomendación personal usar también el editor y los plugins en inglés
